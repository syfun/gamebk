# 项目目标

本项目目标是创建一个基于 web 页面的游戏存档备份和恢复工具。

# 后端需求

- 使用 Go 语言编写后端服务，指定 gin 框架，目录结构为标准 Go 项目布局
- 使用 sqlite 数据库存储配置信息, 使用 jmoiron/sqlx 操作数据库
- 提供 RESTful API 接口
- 支持游戏存档的备份和恢复, 备份时以文件拷贝的方式拷到指定目录下
- 支持多款游戏，可自定义游戏路径和备份路径
- 暂时支持 windows 平台，且以后台服务运行
- 在 mac 上开发，交叉编译完成 windows 平台的可执行文件

# Restful API 接口

- 创建新游戏配置
- 备份游戏存档
- 恢复游戏最新备份
- 恢复游戏指定备份
- 获取游戏配置列表
- 获取游戏备份列表
